第一讲

基本概念

  • 计算机图形学
  • 数字图形处理
  • 计算机动画
  • 计算机视觉
  • 感知与交互

  1. 增强现实技术AR：

    • 融合了三维动画、立体视觉和图像处理；
    • 建模、渲染、位置标定、图像融合；
  2. OpenGL是一种用于创建实时3D图像的编程接口。

  3. 三维这个术语表示一个正在描述或显示的物体具有三维维度：宽度、高度、深度；

    • 计算机3D图形实质上也是平面的；
    • 在计算机屏幕上显示的二维图像，提供深度（或第三维）的错觉；
    • 2D+透视 = 3D; 透视使人产生深度的错觉。

  4. 真正的3D是通过人的两只眼睛观察同一个物体，在视网膜上生成具有视差的两个图像，在人的大脑产生真正的3D视觉感受。

  5. 名词解释：渲染(Rendering)，将数学和图形数据转换成3D空间图像的操作。
  6. 变换(transformation):平移、旋转和缩放
  7. 投影(Projection)：将3D坐标转换成二维坐标。
  8. 光栅化：使用像素进行图形填充。
  9. 纹理贴图：纹理是一幅贴到三角形或多边形上的图片。

  10. 视口映射：

    • 把绘图坐标映射到窗口坐标；
    • 从逻辑笛卡尔坐标映射到物理屏幕像素坐标；
    • 视口就是窗口内部用于绘制裁剪区域的客户区域；视口不一定就是整个窗口，逻辑坐标的每个单位不一定与屏幕像素一一对应。

  11. 投影：从3D到2D，两种投影方式：正投影和透视投影。

    • 正投影：也叫平行投影；特点：没有畸变，视觉不真实；主要用于建筑设计、CAD、2D绘图；
    • 透视投影：近大远小、视觉真实。

  12. 着色(shading)：本质概念；绘制物体颜色

    • Flat
    • Smooth
      • Gouraud
      • Phong
    • flat shading:
      • constant surface shading 单一颜色
    • Gouraud shading:
      • color interpolation shading 颜色插值
    • Phong shading:
      • vertex normal interpolation shading 法向插值

  13. 纹理贴图(Textture Mapping)

    • 纹理是一幅贴到三角形或多边形上的图片
    • 纹理坐标

  14. 混合(Blending)

  15. 总结

    • 3D计算机推行就是讲顶点连接起来，然后得到三角形光栅化
    • 这个过程包括了：变换、着色、纹理和混合。。。

  16. 着色器(shader)

    • 可编程着色器(Programmable Shading)
    • 固定管线与可编程着色器
    • 高度并行
    • 实现前所未有的实时逼真性

  17. 什么是OpenGL？

    • Open Graphics Library； 定义：图形硬件的一种软件接口；最初由SGI创建，用于在不同硬件体系的图形设备上进行二维、三维图形的绘制。OpenGL不是编程语言，特点是并行浮点运算速度极快，但不能进行流程控制。
  18. ARB：标准扩展；EXT：多家支持的扩展；
  19. 背面剔除：画家算法：将需要绘制的图元排序，首先绘制最远的，依次靠近。

使用OpenGL

  • 介绍
    • openGL不负责窗口管理、用户交互和文件读写
    • 在不同的操作系统平台需要调用本地API获取OpenGL绘制上下文(Context)
    • 操作系统绑定： WGL、GLX、CGL
    • 跨平台的工具箱：GLUT = OpenGL Utility Toolkit
      • 兼容Win/mac
      • 最新开源的freeglut
    • 处理扩展的包装库: GLEW: The OpenGL Extention Wrangler Library
    • 一站式的封装：GLTools
  • OpenGL状态机
    • OpenGL使用了一组状态变量的集合来保持图形渲染管线的状态
    • OpenGL使用了一种状态模型(状态机)来跟踪所有状态变量
    • 当值被设置后，就一直保持这个状态
  • OpenGL状态

动画

  • 什么是动画
    • 制作层面：不用真人，用技术手段产生的活动影像
    • 技术层面：动画是一系列静态图像的顺序播放；利用人的视觉残留，产生连续运动的观赏效果
  • 动画分类：1.传统动画：手绘动画 2. 定格动画：黏土动画，木偶动画 3. 计算机：二维动画，三维动画
  • 传统动画制作流程
    1. 创建剧本
    2. 创建主要角色
    3. 绘制故事版(story board)
    4. 高级动画师绘制关键帧(key frame)
    5. 初级动画绘制中间帧（in-between）
    6. 使用摄像机逐个图片拍摄
    7. 使用放映机按照一定帧率播放
  • 定格动画
    1. 创建剧本
    2. 创建主要角色：使用黏土制作模型
    3. 绘制故事版(story board)
    4. 布置场景
    5. 将角色模型摆成各种姿势
    6. 使用摄像机逐个图片拍摄
    7. 使用放映机按照一定帧率播放
  • 二维计算机动画
    1. 解决传统动画的问题
      1. 画面颜色的填充
      2. 中间帧的自动生成
      3. 更低的拍摄成本：更短的时间，不使用底片
    2. 常见的二维动画工具
      1. Adobe Flash
      2. 针对矢量图形进行优化
  • 三维计算机动画
    1. 与二维动画完全不同的制作方式，更接近与定格动画的制作
    2. 三维动画的制作流程
      • 角色和环境建模
        • 创建三维动画和纹理映射
      • 创建动画
      • 渲染输出
    3. 常见的三维动画制作工具
      • Autodesk Maya/3DS Max/SoftImage XSI
      • Motion Builder/Endophin
  • 计算机动画的技术原理
    • 动画有多幅画面组成，当画面快速、连续地播放时，由于人类眼睛存在“视觉滞留效应”而产生动感。停留时间为 1/10 s.
    • 基本概念： 动作的变化是动画的本质。
  • 变形动画 vs 骨骼动画
    • 变形动画优点：1.精细地控制形状的变化 2. 针对面部表情动画适用
    • 变形动画缺点：数据量大，制作复杂度高，很难重用
    • 骨骼动画技术：
      1. 模型创建
      2. 骨骼创建
      3. 骨骼绑定(Rigging)
      4. 动画设定
        • 关键帧
        • 动作数据
    • 三维动画的控制技术
      1. 过程式动画
        • 运动学动画: s=vt
        • 动力学动画: f=ma
      2. 关键帧动画
        • 动画师指定少量关键帧的骨骼动画
        • 计算机通过指定插值算法计算中间的姿势
      3. 动作数据驱动的角色动画
        • 使用动作捕捉技术获取动作数据
        • 将动作数据应用到骨骼动画中
      4. 表演动画
        • 使用真实演员实时控制动画角色
  • 动画技术细节
    1. 关键帧动画
      • 使用关键帧的动画
        1. 肢体动画
        2. 面部表情
        3. 头发和衣物
        4. 光源和摄像机
      • 如何制定关键帧
        • 位置(Position)和朝向(Orientation)
    2. 关键帧动画的插值技术
      • 线性插值技术
        • Linear interpolation
        • 优点：实现简单、计算快速
        • 缺点：运动单一、过渡生硬
      • 非线性插值技术
        • Cubic curve interpolation
        • 优点：过渡运动连续
  • 朝向的插值
    • 朝向的表示
      • 使用矩阵表示 1. 容易使用计算机处理 2. 不可插值
      • 使用欧拉角： 1.表示简单直接 2. 万向锁死锁
      • 使用四元数：球面线性插值
    • 欧拉角：左手定则
      • Roll 翻滚角；
      • Yaw 偏航角；
      • Pitch 俯仰角
    • 欧拉角问题
      • 每次旋转都在前一次的改变的局部坐标系中进行
      • 旋转结果与顺序相关：XYZ、ZXY
      • 插值的困难：0-360
      • 万向锁(Gimbal Lock)
    • 四元数：四维空间
      • 复数的一种扩展
        • 由4个实数组成
        • s,x,y,z或[s,V] s是标量，V是矢量
        • 形式： s + xi + yj +zk
        • i,j,k关系：
    • 四元数运算法则
      • 加法：
      • 乘法表：
        ij=k, ij=-k
    • 表达
    • 特点
      • 包含了足够价值的朝向信息；操作：合成和插值
      • 旋转的合成可利用四元数乘法实现：
        • Q= Q1*Q2
        • 先做Q2的旋转，再做Q1的旋转的结果
        • 多个四元数的旋转也是可以合并的
        • 两个四元数做一次乘法需要16次乘法和加法
        • 3*3的矩阵则需要27次运算
    • 四元数的插值
      • 四元数是一个可以规范化的4维向量
      • 最简单的插值算法就是线性插值
      • 球形线性插值
    • 四元数的缺点
      • 存储多余的动画数据
      • 四元数不合法
      • 难以使用
    • 四元数的优点
      • 不存在万向节锁的问题
      • 平滑插值
    • 四元数、欧拉角和旋转矩阵的转换

    • 方案选择
      • 人机交互：欧拉角
      • 大量的存储：欧拉角或四元数
      • 插值：四元数
      • 运动叠加：四元数或旋转矩阵
      • 图形绘制：旋转矩阵
  • 动作捕捉： 使用设备记录人体的三维动作信息
